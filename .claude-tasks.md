- **[FIX]** `npm run build` hang on Windows — vite build never exits, SSR build never runs
  - Files changed: `vite.config.js`
  - Details: `vite build` completed (`✓ built`) but the node process never exited, so `&& vite build --ssr` never started and the SSR bundle went stale. Diagnosed via `process.getActiveResourcesInfo()`: after the bundle closes, the esbuild child process (ProcessWrap) leaves a pipe (PipeWrap) open, keeping the event loop alive (vite 7.0.6 / esbuild 0.25.8 / Node 22.22 on Windows). Not caused by laravel-vite-plugin (react-only build hung too); no SCSS involved. Fix: `forceExitAfterBuild` plugin in vite.config.js — `closeBundle` (sequential, order post) calls `process.exit(0)` after a 500ms grace. Full `npm run build` now runs both passes and exits 0.

## 2026-07-09 — Admin building/website flow overhaul (client feedback)

**Feedback (paraphrased):** Building creation is too slow because taxonomies (neighbourhoods, sub-neighbourhoods) and developers must be created beforehand; the manual Price Range input should come from MLS instead; a CSV import with field mapping is needed; developer add/edit should be possible inline; the building landing-page (website) flow should be 2 steps and domain connect wasn't working.

**Root causes:**
- Building Create/Edit forms only offered static dropdowns for neighbourhood/sub-neighbourhood/developer — no inline creation, forcing a round-trip to 3 other admin pages first.
- `price_range` was a free-text input even though live MLS (Repliers) data is available per building address.
- No bulk import existed at all.
- Sub-neighbourhood filter on the Create form compared string select values to numeric IDs with `===`, so filtering by neighbourhood always returned an empty list (latent bug).
- Website creation was already 2 steps; domain connect fails because PLOI_API_TOKEN / PLOI_SERVER_ID / PLOI_SITE_ID are not set in the environment — the UI just didn't say so up front.

**Fix:**
- New JSON quick-create endpoints: `POST admin/api/neighbourhoods`, `POST admin/api/sub-neighbourhoods`, `POST admin/api/developers` (firstOrCreate by name).
- New reusable `QuickCreateSelect` component (select + inline "+ New" form) used for Developer, Neighbourhood, and Sub-Neighbourhood on both Building Create and Edit pages.
- Removed the manual Price Range input from both forms; `RefreshBuildingPriceRangeJob` (dispatchAfterResponse) recomputes `price_range` from live Repliers sale listings after every building save via `Building::refreshPriceRangeFromMls()`.
- CSV import wizard at `admin/buildings/import` (upload → column-to-field mapping with auto-guess → results) backed by `BuildingImportController` + `BuildingCsvImportService`; auto-creates developers/neighbourhoods/sub-neighbourhoods by name, skip-or-update duplicate handling, per-row error reporting. "Import CSV" button added to Buildings index.
- Fixed the string/number `===` sub-neighbourhood filter bug.
- Websites Create: amber warning under the Custom Domain field when Ploi auto-provision is not configured, explaining exactly why the domain won't connect and what to set.
